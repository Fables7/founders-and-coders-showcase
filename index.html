Styling

When I initially started incorporating styling into my web and mobile apps, I began by using React's stylesheet. Although it was a valid approach for applying styles, I found it cumbersome as it increased the size of each file and made the code appear disorganized. To address this, I adopted the use of CSS modules. By utilizing separate files for each CSS module, I could scope styles to individual components, resulting in cleaner and more organized code. However, this approach limited me to plain CSS, lacking programmable features like loops and functions. To overcome this limitation, I transitioned to CSS-in-JS libraries like styled-components. I greatly preferred this method as it empowered me with the full capabilities of JavaScript, enabling programmable features and dynamic styles based on state. More recently, while developing web apps with React and Next.js, I discovered a utility class library called tailwindcss. Utilizing tailwindcss provided several benefits. It allowed me to write styling using utility classes in a much shorter and more efficient manner compared to regular plain CSS. Additionally, it automatically purges any unused CSS, resulting in highly optimized bundle sizes.
